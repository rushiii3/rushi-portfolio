# Content Blueprint

Use this file for deterministic section assembly across long-form outputs.

## Required Assembly Order (Long-Form)

1. `title`
2. `meta_title`
3. `meta_description`
4. `tldr` (40-70 words)
5. `introduction_hook`
6. `definition_block`
7. `core_sections` (H2/H3 equivalent)
8. `comparison_table` (only if useful)
9. `actionable_framework`
10. `common_mistakes`
11. `faq` (5 questions exactly)
12. `conclusion`
13. `citations`
14. `internal_link_suggestions` (3)

## Intent Mapping

- Informational:
  - Emphasize explanation, definitions, and examples.
  - CTA should be soft (learn more / next step).
- Commercial:
  - Emphasize comparison criteria, tradeoffs, and buyer questions.
  - Include neutral evaluation framework.
- Transactional:
  - Emphasize action steps, implementation prerequisites, and decision checklist.
  - Keep CTA direct but factual.
- Navigational:
  - Emphasize fastest path to destination resource/page.
  - Minimize conceptual detours.

## Content Depth Rules

- Standard: concise treatment, limited branching.
- Long: deeper examples and expanded FAQ detail.
- Authority: include frameworks, edge cases, and caveats.
- Pillar: comprehensive coverage, internal link map, and broad entity coverage.

## Definition Block Pattern

Format:

- one-sentence definition
- when to use
- when not to use
- example in context

## Actionable Framework Pattern

Use a named sequence:

1. Diagnose
2. Plan
3. Execute
4. Measure
5. Iterate

Each step should include:

- objective
- concrete action
- expected output
- common failure mode

## Common Mistakes Pattern

For each mistake:

- why it hurts performance/clarity
- how to detect it quickly
- corrective action
